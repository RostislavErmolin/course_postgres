# Задание 1: развертывание Postgres

1. Создать репозиторий на https://github.com. Склонировать его в локальную директорию.
2. Установить и запустить кластер postgres с помощью docker compose
3. Используя утилиту psql подключиться к postgres, и проверить доступность субд.
   Для этого попробуем ответить на вопрос, сколько баз данных доступно в postgres сразу после развертывания
   кластера. Необходимо вывести их имена.
4. Синхронизировать изменения с удаленным репозиторием.

## Инструкции

1. Авторизуйтесь на гитхабе и создайте публичный репозиторий. При создании выберите Add README.
2. Склонируйте репозиторий в рабочую директорию. Предварительно лучше создать ssh ключ согласно [инструкции](https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent).
и добавить его в свой аккаунт.
3. В рабочей директории необходимо создать файл compose.yaml со следующим содержанием
```yaml
services:
  db:
    image: postgres:18.3-bookworm
    container_name: db
    restart: always
    # set shared memory limit since it is needed for postgres
    shm_size: 128mb
    ports:
      - "127.0.0.1:5432:5432"
    volumes:
      - ./postgresql_data:/var/lib/postgresql
    environment:
      POSTGRES_PASSWORD: example

```
В процессе первоначального развертывания будет создана учетная запись пользователя с именем postgres.
Переменная POSTGRES_PASSWORD позволяет задать пароль для этого пользователя.
Рекомендуется заменить ее значение на что-то больше похожее на пароль.

4. В терминале перейдите в директорию проекта и выполните команду
```shell
docker compose up -d
```
Для выполнения этой и последующих команд может потребоваться sudo. Рекомендуется выполнить инструкции https://docs.docker.com/engine/install/linux-postinstall/, чтоб можно было выполнять команды без sudo.

Данная команда инициирует скачивание докер образа postgres и запустит postgres в docker контейнере. Если точнее,
то эта команда запустит все сервисы из файла compose.yaml, но поскольку сейчас у нас там один
сервис с постгрес, то запустится только он. Можно запустить конкретно postgres,
указав имя сервиса db в конце команды.
```shell
docker compose up -d db
```
Вы можете проверить, что субд работает с помощью команды
```shell
docker ps
```
Данная команда показывает список всех запущенных контейнеров.
Из результатов можно увидеть, что контейнер с postgres запущен, а также хост и порт, по которым он доступен.

Также можно заметить, что в директории проекта создалась поддиректория postgresql_data. Это так называемый
docker volume --- директория, которая доступна и в контейнере, и в хост системе. Волюм нужен, чтоб обеспечить
персистентность данных, когда процесс запускается в контейнере. Сам контейнер при этом является эфемерным,
т.е. его запросто можно удалить и создать заново. Если бы не было волюма, то удаление контейнера привело бы
к удалению данных в postgres.

Для того, чтоб остановить все контейнеры из compose.yaml нужно выполнить команду
```shell
docker compose stop
```
либо
```shell
docker compose down
```
Разница между этими двумя командами в том, что последняя не просто тушит сервисы, но также и удаляет
ресурсы, связанные с ними: контейнеры и внутренние сети. Обе команды можно выполнить с указанием
конкретного сервиса
```shell
docker compose stop db
```

5. Теперь проверим, что наша база данных действительно функционирует. Для этого будем использовать клиент
   psql, который запустим в терминале. Для того, чтоб подключиться к кластеру postgres необходимо выполнить
```shell
psql -h {адрес хоста} -p {порт} -U {имя пользователя}
```
Из вывода `docker ps` мы уже знаем адрес и порт контейнера с postgres. Кстати порт 5432
используется в psql по умолчанию, так что его можно не указывать.

Выполнив вход в psql мы окажемся в командной оболочке этого клиента. При этом начало командной строки
поменяется на `postgres=#`. Здесь можно выполнять как команды, специфичные для psql (метакоманды), так и SQL. psql
команды начинаются с символа `\`.

Самая полезная команда psql --- это команда помощи `\?`. Набрав ее, можно получить список всех доступных
команд. По-умолчанию список команда выводится в пейджере less. Ваша задача найти в списке команд ту, которая
выводит все доступные базы данных (см. памятку ниже).

Вторая полезная команда psql --- это команда выхода `\q`.

Также вы всегда можете посмотреть подробную справку со всеми метакомандами набрав в терминале
```shell
man psql
````
Справка будет доступна в пейджере less.

## Памятка по пейджеру less
### Навигация
* пробел --- страница вниз
* b --- страница вверх
* enter --- строка вниз
* стрелки --- построчно
* q --- выйти
* h --- справка
### Прыжки
* g --- начало
* G --- конец
* 50% --- середина (можно любой процент)
### Поиск
* / - поиск вперёд. Набираешь /keyword и Enter - найдёт команды, где есть слово keyword
* n - следующее совпадение
* N - предыдущее совпадение
* ? - поиск назад

6. Не забываем зафиксировать изменения и синхронизировать их с удаленным репозиторием в конце работы.
